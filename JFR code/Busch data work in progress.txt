Keep them separate so I can rerun imports without recomputing bid curves, and rerun bid curves without rebuilding all tables.

## Required changes to script 3 (loader)

### Scope
`3_import_k_means_csv_to_sqlite.py` should read:
1. `undiscounted_contracts_*.csv`
2. `k_means_carbon_removal_contract_years_*_assignments.csv`
3. `k_means_carbon_removal_contract_years_*_centers.csv`
4. `k_means_carbon_removal_contract_years_*_summary.csv`

### Table build rules
1. Build `Pixel_bids` from `undiscounted_contracts_*.csv`.
	- `pixel_id <- pixel_id`
	- `selected_option <- option`
	- `contract_years <- best_contract_length`
	- `area_ha <- area_ha`
	- `NPV_0_pct_per_ha <- bid_r000`
	- `NPV_015_pct_per_ha <- bid_r015`
	- `NPV_03_pct_per_ha <- bid_r030`
	- `NPV_06_pct_per_ha <- bid_r060`

2. Build one carbon table per contract length:
	- `cluster_carbon_schedules_{years}`
	- source: centers CSV (`cluster_id`, `year_1..year_T`)
	- output rows: `(cluster_id, growth_year, tC_per_ha_per_year)`

3. Update `Pixel_bids.cluster_id` from assignments CSV:
	- assignments columns: `contract_years`, `pixel_id`, `cluster_id`
	- join key: `(pixel_id, contract_years)`
	- rule: every imported pixel row must receive one cluster_id

4. Table lifecycle rule:
	- clear and rebuild `Pixel_bids`
	- drop/recreate `cluster_carbon_schedules_{years}` tables for all years present
	- this avoids stale rows from prior runs

### Verification checks in script 3
1. For each `{years}`, assignment row count equals `Pixel_bids` row count where `contract_years={years}`.
2. For each `{years}`, distinct `cluster_id` count in centers equals summary equals assignments.
3. For each `{years}`, all assignment `pixel_id` values exist in `Pixel_bids` for the same `contract_years`.
4. In `Pixel_bids`, `(pixel_id, contract_years)` is unique.
5. After assignment update, no null `cluster_id` remains in `Pixel_bids`.

## Required changes to script 4 (bid curves)

### Scope
`4_move_Busch_data_to_SMDAMAGE.py` should read `Pixel_bids` and write `cluster_forestry_bid_curves_{years}`.

### Core logic
1. Do not recompute NPV from establishment cost or harvest-value components.
2. Use `NPV_*_per_ha` from `Pixel_bids` as authoritative cost inputs.
3. For each `years` and each `cluster_id`, sort pixels by NPV and bucket by area share.
4. Keep current default rule:
	- bucket size: 3% of area
	- omit top-cost tail: 1% of area
5. Write rows as:
	- `cluster_id`
	- `discount_rate_00` in `{0, 15, 30, 60}`
	- `bid_step`
	- `npv_max_per_ha`
	- `step_area_ha`

### Run entrypoint
Add one main run path that:
1. discovers contract years from `Pixel_bids`
2. rebuilds `cluster_forestry_bid_curves_{years}`
3. logs row counts by `(years, discount_rate_00)`
4. verifies monotonicity per curve (non-decreasing `npv_max_per_ha` by `bid_step`)
